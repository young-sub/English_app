# Repository Work Policy

## Mandatory workflow for feature updates

For any feature update in this repository, follow this order:

1. Implement the feature update.
2. Run relevant verification (at minimum: related tests; plus build/type checks when applicable).
3. Fix issues until verification passes.
4. Create a Git commit as the final step with a clear message.

## Notes

- This policy applies to functional/code updates.
- Commit only after successful verification.
